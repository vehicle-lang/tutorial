\begin{frame}{Vehicle}

\begin{itemize}
\item A tool for the whole life-cycle of a neural network verification property:
\begin{itemize}
\item Training
\item Verification
\item Integration
\end{itemize}

\vspace{1em}

\item A pure functional specification language provides the glue.

\vspace{1em}

\item Improving trust in AI systems using the power of functional programming!
\end{itemize}
\end{frame}


\begin{frame}{Possible applications}

\begin{columns}
\begin{column}{0.5\textwidth}
Robustness of:
\begin{itemize}
\item Intrusion detection systems
\item Malware detection systems
\item Chatbot systems
\item Sensor fusion pipelines
\end{itemize}
\end{column}

\begin{column}{0.5\textwidth}

\begin{itemize}

\item Conservation laws
\end{itemize}

\vspace{2em}

\begin{itemize}
\item Autonomous vehicles
\item Network traffic balancing
\end{itemize}
\end{column}
\end{columns}

\end{frame}


\begin{frame}{Conclusions}

Functional programming takeaways:
\begin{itemize}
	\item Expressive type systems (generalisation, instance resolution) are especially useful for multi-backend systems.
	\item Possibly more useful in the backends than user code!
\end{itemize}

\vspace{1em}

PL/verification research challenges:
\begin{itemize}
	\item Are type systems for abstract interpretation-based verifiers possible?
	\item A moderately performant formally verified verifier would be amazing!
\end{itemize}

We're always interested in collaborations so please reach out!
